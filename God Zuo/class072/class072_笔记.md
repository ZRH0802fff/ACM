# class072 最长递增子序列问题与拓展

## 核心知识

### 最长递增子序列 (LIS) 最优解 — O(n log n)

**核心数据结构：`ends` 数组**
- `ends[i]` = 目前发现的所有长度为 `i+1` 的递增子序列中，**最小的结尾值**
- `ends` 数组**严格升序**（证明：若长度为6的LIS结尾为21，则其前一个数必<21，所以长度为5的LIS最小结尾不可能≥21）

**更新流程：**
1. 遍历每个数 `x`
2. 在 `ends[0..len-1]` 中二分找 `>= x` 的最左位置
3. **找到了** → 更新该位置为 `x`（更小的结尾更优）
4. **没找到** → 扩充有效区 `ends[len++] = x`（冲到了新长度）

**关键二分（class072 核心）：**
| 问题 | 二分策略 | 函数 |
|------|---------|------|
| 严格递增子序列 | `>= num` 最左 | `bs1` |
| 不下降子序列 | `> num` 最左 | `bs2` |
| 不上升子序列（右→左） | `< num` 最左 | `bs1` (P8776) |

**重要澄清：** `ends` 数组的最终状态**不代表**任何一个实际的LIS，它只是每个长度的最小结尾信息的集合。

## 题目

### 01 最长递增子序列 — LeetCode 300

**思路演进：**
- **O(n²) DP**：`dp[i]` = 以 i 结尾的LIS长度，向左扫描比它小的数
- **O(n log n)**：`ends` 数组 + 二分加速

**核心点：** `ends` 的严格升序性保证了二分正确性。最终答案 = `len`（有效区长度），不需要单独维护 dp 数组。

**坑点：** `ends` 与 `std::ends` 冲突，C++ 中必须改名（如 `edp`）。

---

### 02 俄罗斯套娃信封 — LeetCode 354

**核心技巧——排序策略：**
- **宽度**从小到大
- **宽度相同时，高度从大到小**

**为什么高度要降序？** 防止相同宽度的信封被套在一起。如果高度升序，如 `(2,4)` `(2,9)`，在高度数组 `[4,9]` 中会形成长度为2的LIS，但实际上宽度相同的信封不能互套。

**正确性：** 排序后求高度LIS，只会找到「宽度更小 且 高度更小」的信封组合。

**坑点：** 排序比较器要正确处理 `a[0]==b[0]` 时高度降序，否则同宽信封会被误套。

---

### 03 使数组K递增 — LeetCode 2111

**核心洞察——分组独立：**
- 按 `i % k` 分成 k 组，每组之间无关
- `arr[i-k] <= arr[i]` 等价于每组内不下降

**解法：** 每组长度 - 该组最长不下降子序列长度 = 该组最少修改次数，各组累加。

**坑点：** 是**不下降**不是严格递增，二分要用 `> num` 最左（bs2），不要用错。

### 04 最长数对链 — LeetCode 646

**解法一：LIS + 分离技巧**
- 按开始位置排序
- **查**：用 `pair[0]`（开头）在 `ends` 中查 `>=` 最左
- **改**：用 `pair[1]`（结尾）更新 `ends`，且**只有真的更小才改**

**解法二：贪心**
- 按结尾位置排序
- 每次选能接上的（开头 > 上一个结尾）就接上
- 更简洁，O(n log n)，最优

**核心点（分离技巧）：** 查的东西（开头）≠ 入 ends 的东西（结尾），且更新有决策（`min(ends[find], pair[1])`）。

**坑点：** LIS解法中 `ends[find] = min(ends[find], pair[1])`，必须取 min，不是直接覆盖。

### 05 修改K个连续值 — Luogu P8776

**核心洞察——紧贴性：** 修改的 K 个连续值，一定紧挨着某个被选入最终子序列的数，不需要中间有空隙（因为空出来的元素反正不要，K个紧贴着刷同样不下降）。

**枚举框架：**
```
对每个分界点 j (第一个不被修改的位置):
  ans = max(ans, left + K + rightArr[j])
  其中 left  = 左侧不下降子序列长度（结尾 ≤ arr[j]）
       rightArr[j] = 必须以 arr[j] 开头的右侧不下降子序列长度
```

**分离技巧（滑动窗口）：**
- **查 ends**：当前分界值 `arr[j]`（决定左侧能取多少）
- **入 ends**：滑出窗口的 `arr[i]`（新暴露的左边界）

**rightArr 求解——反向最长不上升：**
- 从右往左遍历
- `rightArr[i]` = 以 `arr[i]` **开头**的最长不下降子序列
- = 从右往左看，以 `arr[i]` **结尾**的最长**不上升**子序列
- 二分：`< num` 最左（bs1）

**坑点：**
1. 最后一种情况：K 个在末尾，右侧为空，`ans = max(ans, len + K)`
2. ends 在 `computeRight()` 和主循环中复用，注意两次使用的二分策略不同（bs1 vs bs2）
3. `rightArr` 的含义要理解清楚：一定以该位置**开头**的后缀LNDS长度

## 技巧总结

| 技巧 | 出现题目 |
|------|---------|
| ends 数组 + 二分加速 | 01, 02, 03 |
| 排序策略消除一维 | 02（宽度升+高度降） |
| 分组独立 | 03（按 mod k） |
| 查改分离 | 04（查开头改结尾）, 05（查分界入滑出） |
| 反向遍历等价转换 | 05（右→左 不上升 = 左→右 不下降） |
| 贪心替代DP | 04（按结尾排序贪心选） |
